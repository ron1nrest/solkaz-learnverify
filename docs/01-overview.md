# 01. Обзор проекта

## Название

**SolKaz LearnVerify** (рабочее имя CLI: `learnverify`)

## Одно предложение

Open-source CLI и спецификация заданий, которые **автоматически проверяют выполнение практики Solana на Devnet** по публичному адресу кошелька студента.

## Проблема

| Кто | Боль |
|-----|------|
| **Студент** | Неясно, зачтут ли работу; стыдно спрашивать; скриншоты не дают обратной связи по шагам |
| **Ментор** | 2–4 часа в неделю на ручную проверку 20–40 кошельков в Explorer |
| **Bootcamp KZ** | Нет стандартизированного «автотеста» для Devnet-домашек на русском языке |
| **Экосистема** | Много AI-агентов для DeFi, мало **инфраструктуры для образования** |

## Решение

1. Ментор (или open-source пакет) публикует **`challenge.yaml`** — список проверяемых on-chain условий.
2. Студент выполняет задания в **Devnet** своим кошельком.
3. **`learnverify verify`** запрашивает данные у Solana RPC, сверяет условия, печатает отчёт.
4. **`learnverify cohort`** прогоняет ту же проверку по списку кошельков для группы.

Доказательство навыка — **сама цепочка**, а не скриншот.

## Целевая аудитория

| Сегмент | Использование |
|---------|----------------|
| Студенты вузов / bootcamp KZ | Самопроверка перед сдачей |
| Менторы Superteam Kazakhstan, AlmaU | Массовая проверка когорты |
| Преподаватели | Готовые недельные challenge-пакеты |
| Open-source сообщество | Форк challenge'ов под свой курс |

## Ценность (value proposition)

### Для студентов
- Мгновенная обратная связь: что сделано, что осталось.
- Честная оценка: нельзя «нарисовать» скриншот.
- Практика в реальном стеке: кошелёк, транзакции, Explorer.

### Для менторов
- Проверка группы за **минуты** вместо часов.
- Единые критерии для всех участников.
- Отчёт: кто отстаёт и на каком `task`.

### Для гранта Solana Foundation Kazakhstan
- **Education:** готовый учебный трек + RU документация.
- **Developer tooling:** CLI + declarative spec + (post-MVP) GitHub Action.
- **Censorship resistance:** верификация через консенсус 2+ RPC (см. [08-security.md](08-security.md)).
- **Public good:** MIT, без платных API для базового сценария.
- **Regional:** пилот на когорте в Казахстане, материалы на русском.

## Границы продукта (non-goals)

LearnVerify **не является**:

- AI-генератором кода или «Solana ChatGPT»;
- кошельком или custody-сервисом;
- инструментом для mainnet и реальных денег (в MVP);
- заменой лекций и теории — только **проверка практики**;
- конкурентом [Solana Agent Kit](https://github.com/sendaifun/solana-agent-kit) — другой слой (education verification).

## Позиционирование для ревьюеров гранта

> **LearnVerify — это автотесты для Solana-домашек на Devnet: open-source инфраструктура для bootcamp'ов Казахстана, а не ещё один AI agent.**

## Связь с экосистемой

| Ресурс | Связь |
|--------|--------|
| [Solana Foundation Curriculum](https://github.com/solana-foundation/curriculum) | Challenge'ы MVP выровнены по темам недель 1–2 |
| Superteam KZ bootcamp / AlmaU | Целевой пилот (см. [07-grant-playbook.md](07-grant-playbook.md)) |
| Solana U | Дополнительный канал дистрибуции challenge-пакетов |
| Helius / публичные RPC | Read-only доступ; free tier достаточен для пилота |

## Метрики успеха (MVP + пилот)

| Метрика | Цель MVP |
|---------|----------|
| Challenge-файлов в репо | ≥ 5 (к концу W3 — 8) |
| Типов проверок (`task types`) | ≥ 4 |
| Успешных `verify` на тестовых кошельках | 100% на golden fixtures |
| Пилотная когорта | ≥ 10 студентов |
| Время проверки 1 кошелька | < 15 с (типичный Devnet) |
| Время проверки 30 кошельков (`cohort`) | < 3 мин |
| Документация | RU, полный путь «от нуля до verify» |

## Термины

| Термин | Значение |
|--------|----------|
| **Challenge** | Файл `*.yaml` с метаданными и списком `tasks` |
| **Task** | Одно атомарное on-chain условие (баланс, перевод, mint…) |
| **Verify** | Прогон всех `tasks` для одного `wallet` |
| **Cohort** | Пакетная проверка многих кошельков |
| **Golden wallet** | Тестовый кошелёк с заранее известным состоянием Devnet |
